Iterators (ES2015)

				
var someIterableObject = {
	[Symbol.iterator]: function(){
		var i = 1;
		return {
			next: function() {
				if (i <= 42) {
					return {done: false, value: "Chunk " + i++ };
				} else {
					return {done: true};
				}
			}
		}
	}
}

next(): Chunk 1 Chunk 2 Chunk 3 ... Chunk 42 DONE

  • Based on the pattern
  • Came with ES6/2015, Symbol => a way to have special keys in an object, without the fear, some downstream lib has used that key already
  • ES6 Iterables have conventions
  • object with a next function returning {done: false, value: "... "}; or {done: true};
  • available at the Symbol.iterator attribute/key
  • Array, Map, ... , implement it
iteration

Generators (ES2015)

				
var someIterableObject = {
	getChunksGeneratorFunction: function* (){ 	//returns an iterator
		for (var i=1; i <= 42; i++) {
			yield "Chunk " + i;		//next value
		}
		/* can be implicit */return; 		//done
	}
}

Yielding: Chunk 1 Chunk 2 Chunk 3 ... Chunk 42 DONE

  • Much simpler way to implement iterators, is using generators
  • Came with ES6/2015
  • function* indicated it being a generator
  • yield is the next value that is returned
  • Calling a generator function does not execute its body immediately; an iterator object for the function is returned instead.
generators

Working with iterators 

				
var iterator = someIterableObject[Symbol.iterator]();
for (var result = iterator.next(); !result.done; result = iterator.next()) {
	console.log(result.value);
}

for-of loop (ES2015)

				
for (var value of someIterableObject.getChunksGeneratorFunction()){
	console.log(value)
}
  • You can easily use iterators directly
  • But the for-of loop can handle it for you

Async / await (ES2017)

				
function doSomethingAsync(){
	//make some backend call, etc.
	//return a promise
	return Promise.resolve(5)
}

async function doAsyncStuff(url) {
	try {
		var iResolvedValue = await doSomethingAsync();
		return iResolvedValue + 1; //will be wrapped into promise as it is an async function
	}
	catch (error) {
		console.log("ERROR: " + error.stack);
	}
}
  • Simpler way to work with promises and structure source code
  • await oPromise
  • Open "parallel" execution in same control flow
  • async function returns a promise of its return value
  • Still necessary to understand PROMISES!

Let's combine iteration + async/await

Async Iterators (ES2018)

				
var someIterableObject = {
	[Symbol.asyncIterator]: function(){
		var i = 1;
		return {
			next: function() {
				if (i <= 42) {
					return Promise.resolve({done: false, value: "Chunk " + i++ });
				} else {
					return Promise.resolve({done: true});
				}
			}
		}
	}
}
* similar to normal iterator + async * BUT resolves the iterator value * To impress people: declarative pull streams iteration

Async Generators (ES2018)

				
function someAsyncOperation() {
	/* something async */
	Promise.resolve(1);
}

var someIterableObject = {
	getAsyncChunksGeneratorFunction: async function* (){ 	//returns an async iterator
		for (var i=1; i <= 42; i++) {
			var value = await someAsyncOperation();	//next() return now with a promise
			yield "Chunk " + i + value;	//resolves next value after someAsyncOperation
		}
		/* can be implicit */return; 			//done
	}
}
  • Allows Queuing of next() invocations (calling it more often than it resolved already values)
  • BUT resolves the iterator value
iteration

Working with async iterators 

				
var iterator = someIterableObject[Symbol.asyncIterator]();
iterator.next().then(function(result){
	if (!result.done){
		console.log(result.value);
	}
});

Async for-of loop (ES2018)

				
for async (var value of someIterableObject.getAsyncChunksGeneratorFunction()){
	console.log(value)
}
  • You can easily use iterators directly, it will just return promisses
  • But the async for-of loop can handle it for you
